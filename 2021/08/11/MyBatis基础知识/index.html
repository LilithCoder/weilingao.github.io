MyBatis简介

ORM模型

Java程序是通过JDBC(Java Database Connectivity)来连接数据库的

JDBC是Java程序和数据库交互的标准API，API分为面向Java程序开发的API以及面向数据库驱动程序开发的API，Java程序操作JDBC接口(来自Java标准库)，通过JDBC驱动来访问连接数据库的，其中JDBC驱动是JDBC接口的底层支持实现，且具体实现取决于各个不同的数据库厂商

Java程序 -> JDBC接口 -> JDBC驱动 -> 数据库

JDBC让Java代码不再需要针对不同的数据库厂商分别开发，但JDBC的弊端在实际开发中也是显而易见的，通过一个简单的sql进行查询操作需要：

  1. 注册数据库驱动类、指定数据库的信息
  2. 通过DriverManager获取数据库连接
  3. 创建statement对象，执行sql
  4. 返回resultSet，并读取数据，转化为POJO(Plain Ordinary Java Object 简单Java对象)
  5. 关闭所有相关资源

看起来一次查询涉及到很多步骤，特别是第4步关系模型到对象模型的转换相对复杂，为了对jdbc进行封装，ORM模型出现了

ORM(Object Relational Mapping 对象-关系映射)就是数据库的表和POJO的映射关系模型，这样的映射是通过映射配置完成的

MyBatis的特性

MyBatis是一个基于ORM模型的持久层框架：

  1. 可以通过XML或者注解的方式进行配置和映射
  2. 相较于其他的ORM框架，Mybatis将Java方法和sql语句关联起来，开发者可以直接在映射配置文件中编写sql语句，这给了我们直接优化SQL语句的机会，让SQL语句选择合适的索引，能更好地提高系统的性能，比较适合大数据量、高并发等场景
  3. sql语句可以被动态生成
  4. MyBatis还具备缓存机制，当一条SQL语句被标记为可缓存后，首次执行它时从数据库获取的所有数据会被存储在高速缓存中，后面再执行这条语句时就会从高速缓存中读取结果，而不是再次命中数据库

MyBatis的构成

SqlSessionFactoryBuilder

SqlSessionFactoryBuilder是一个构造器，它可以利用xml或者一个预先定制的configuration实例来构建SqlSessionFactory实例，开发时候尽量使用xml配置的方式来创建SqlSessionFactory。通过SqlSessionFactoryBuilder可以构建多个SqlSessionFactory

SqlSessionFactoryBuilder的作用就是构造器，一旦构建了SqlSessionFactory，它就失去了存在的意义，应该将其废弃回收，以保证所有的 XML 解析资源可以被释放给更重要的事情，所以它的生命周期只存在于方法的局部，也就是说他的最佳作用域是方法作用域

SqlSessionFactory

SqlSessionFactory是整个数据库映射关系经过编译后的内存镜像

SqlSessionFactory是一个接口，用来创建SqlSession

SqlSessionFactory 一旦被创建就应该在应用的运行期间一直存在

SqlSessionFactory采用单例模式，每一个数据库只对应一个SqlSessionFactory，其好处就是可以重复使用这唯一对象，且它在内存中读取运行速度比较快

每个基于 MyBatis 的应用都是以一个 SqlSessionFactory 的实例为核心的。SqlSessionFactory 的实例可以通过 SqlSessionFactoryBuilder 获得。

构建SqlSessionFactory的方法有两种，这里只介绍xml方式：
首先创建xml文件输入流，这里信息包括了数据库连接实例的数据源（DataSource）以及决定事务作用域和控制方式的事务管理器（TransactionManager），mybatis的解析程序会将mybatis-config.xml解析到configuration类对象里面（configuration类对象存在于mybatis整个生命周期里，且保存在内存里，方便反复使用），然后利用SqlSessionFactoryBuilder读取configuration类对象创建SqlSessionFactory

SqlSession

SqlSession 既可以发送sql去执行并返回结果，也可以获取mapper接口

SqlSession类似于jdbc中的connection接口对象，每次用完都要在finally里正常关闭它

每个线程都应该有它自己的 SqlSession 实例。SqlSession 的实例不是线程安全的，因此是不能被共享的，所以它的最佳的作用域是请求或方法作用域。
