MyBatis简介

ORM模型

Java程序是通过JDBC(Java Database Connectivity)来连接数据库的

JDBC是Java程序和数据库交互的标准API，API分为面向Java程序开发的API以及面向数据库驱动程序开发的API，Java程序操作JDBC接口(来自Java标准库)，通过JDBC驱动来访问连接数据库的，其中JDBC驱动是JDBC接口的底层支持实现，且具体实现取决于各个不同的数据库厂商

Java程序 -> JDBC接口 -> JDBC驱动 -> 数据库

JDBC让Java代码不再需要针对不同的数据库厂商分别开发，但JDBC的弊端在实际开发中也是显而易见的，通过一个简单的sql进行查询操作需要：

  1. 注册数据库驱动类、指定数据库的信息
  2. 通过DriverManager获取数据库连接
  3. 创建statement对象，执行sql
  4. 返回resultSet，并读取数据，转化为POJO(Plain Ordinary Java Object 简单Java对象)
  5. 关闭所有相关资源

看起来一次查询涉及到很多步骤，特别是第4步关系模型到对象模型的转换相对复杂，为了对jdbc进行封装，ORM模型出现了

ORM(Object Relational Mapping 对象-关系映射)就是数据库的表和POJO的映射关系模型，这样的映射是通过映射配置完成的
